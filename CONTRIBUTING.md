## Branch Naming
Use these prefixes:
- `fix/` for bug fixes
- `feature/` for new features
- `chore/` for maintenance

## Workflow
1. Create a branch from `main`
2. Make commits using `git commit`
3. Push and create a PR
4. One PR = one logical change
5. Squash on merge
